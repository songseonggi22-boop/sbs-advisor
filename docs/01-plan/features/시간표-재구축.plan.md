## 1. Overview

### 1.1 Purpose

`sbs시간표 최신화/` 폴더의 모든 XLS 시간표 파일에서 강의 데이터를 빠짐없이 읽어  
앱 검색에서 291개 이상의 평일 강의를 전부 표시한다.

### 1.2 Background

**현재 구조 분석 결과:**

| 항목 | 현황 |
|------|------|
| `sbs시간표 최신화/` 파일 수 | 9개 (`[컴퓨터대전]강의시간표 (1)~(8).xls` + `.xls`) |
| `SBS평일/` vs `sbs시간표 최신화/` | MD5 해시 100% 일치 — 동일 파일 |
| 파서 누락률 | **0%** — 모든 셀 정상 파싱 중 |
| 평일 raw 레코드 | 369개 → 중복제거 후 291개 unique |
| `SBS주말/` 파일 | 3개, June 2026까지만 커버 (불완전) |
| 앱 읽기 경로 | `SBS평일/` + `SBS주말/` (두 폴더 분리) |

**누락 원인 추정:**
1. **캐시 미갱신**: `@st.cache_data` 로 인해 파일 업데이트 후 앱을 재시작하지 않으면 구 데이터 유지
2. **데이터 소스 혼재**: 사용자는 `sbs시간표 최신화/`를 기준으로 생각하지만 앱은 `SBS평일/`을 읽음
3. **주말 데이터 미비**: `SBS주말/`이 June 2026 이후 데이터 없음

### 1.3 Related Documents

- 현재 구현: `pages/시간표_생성.py`
- 기존 계획: `TIMETABLE_PLAN.md`

## 7. Architecture Considerations

### 7.1 Project Level

**Starter** 수준 — 단일 Streamlit 페이지, 파일시스템 직접 읽기

### 7.2 Key Architectural Decisions

| Decision | Selected | Rationale |
|----------|----------|-----------|
| 데이터 소스 | `sbs시간표 최신화/` 단일 폴더 | 사용자가 관리하는 최신화 폴더 = 신뢰할 수 있는 소스 |
| 캐시 전략 | `@st.cache_data` 유지 + manual clear 버튼 추가 | 성능 유지하면서 갱신 필요 시 즉시 클리어 |
| 주말 폴더 | `SBS주말/` optional 유지 | 향후 주말 파일 추가 시 자동 로드 |

### 7.3 구현 파일

```
변경 대상: pages/시간표_생성.py

핵심 변경:
① 3줄 상수 변경 (WEEKDAY_DIR 경로)
② load_all_courses() — WEEKEND_DIR optional guard
③ UI 헤더에 새로고침 버튼 (st.button → load_all_courses.clear())
④ 사이드바 또는 헤더에 로드 현황 텍스트
```
